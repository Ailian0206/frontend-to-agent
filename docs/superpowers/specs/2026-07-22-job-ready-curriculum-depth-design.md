# 岗位级 Agent 教程加深 — 设计规格

- **日期**：2026-07-22  
- **项目**：frontend-to-agent  
- **状态**：已确认（2026-07-22）— 按此执行；实现从 M1 plan 开始  
- **决策方式**：用户关注最终效果；技术默认项由实现侧按推荐落地  

## 1. 结果目标（你真正要验收的）

学完本站后，资深前端应能：

1. **对 JD 说话**：用能力清单证明覆盖 Tool/MCP/RAG/HITL/评估/上线等主线能力，以及选修的平台向深度。  
2. **交得出东西**：至少 1 个接近生产的作品集 Agent（权限、评估集、观测、部署说明齐全）。  
3. **经得起追问**：每条主线能力有反例、失败分类、面试追问与可运行 Lab，而不是概念导览。  

**非目标**：算法训练、从零学编程、成为研究科学家。

## 2. 已锁定产品决策

| 项 | 决策 |
|---|---|
| 岗位方向 | 应用/全栈 Agent 为主；平台/基建为选修 |
| 毕业形态 | 能力清单 + 作品集封顶 |
| 内容形态 | 课 / Lab / 选修 三层 + 作品集 |
| 终局范围 | 做满（分里程碑连续交付） |
| 推进顺序 | 先能力（课+Lab）→ 再作品集 |
| 模型调用 | 本地可跑真 API；CI/自动测试只用 mock |
| 落地策略 | 能力地图驱动分层扩建 |

## 3. 学员看到的结构（结果视角）

侧栏四组：

1. **课程** — 讲清楚「为什么 / 何时 / 坑 / 面试怎么答」  
2. **实验 Lab** — 必须能跑、有验收勾选  
3. **选修** — 平台/基建加深（多租户、网关、编排运行时等）  
4. **作品集** — 期末项目规格、里程碑与验收  

现有 16 章迁入「课程」并加厚；Lab/选修/作品集为新增深度层，不是替换门面。

## 4. 岗位能力地图（主线必须会，选修加分）

### 4.1 主线能力（毕业必过）

| ID | 能力 | 学员必须证明 |
|---|---|---|
| S1 | LLM 控制面 | 结构化输出、Prompt 版本化、边界不进业务代码 |
| S2 | Tool 契约 | Schema、权限、超时、幂等、Dry Run |
| S3 | ReAct / 状态图 | 能画清循环与终止条件，能用 LangGraph 表达 |
| S4 | 流式与 Agent UI | 事件协议、可取消、确认面板，不只吐 token |
| S5 | RAG 工程化 | 切块、权限过滤、Recall@K、引用与拒答 |
| S6 | 记忆与隔离 | thread/checkpoint、跨会话事实、租户隔离意识 |
| S7 | HITL | interrupt/resume、审计、幂等副作用 |
| S8 | 多 Agent | Supervisor/路由，知道何时不该拆 |
| S9 | MCP | 接入、白名单、远程鉴权基线 |
| S10 | 评估与安全 | 轨迹断言、注入红队、失败分类 |
| S11 | 上线与成本 | 鉴权/限流/追踪/预算/降级 |

### 4.2 选修能力（平台向）

| ID | 能力 |
|---|---|
| E1 | Agent 网关与策略（allow/deny/审批） |
| E2 | 多租户与数据面隔离 |
| E3 | 长任务队列 / Worker / 可恢复运行时 |
| E4 | 评估平台与回归门禁（数据集治理） |
| E5 | A2A / 多运行时协作边界 |

每条能力绑定：≥1 节加厚课 + ≥1 个 Lab（主线）或选修文；作品集必须覆盖 S1–S11 的可演示证据。

## 5. 深度标准（什么叫「够深」）

### 5.1 课程章（加厚后）

每章至少包含：

- 决策表（何时用 / 何时不用）  
- ≥1 个真实反例（坏设计对照）  
- ≥1 个可运行代码块（含依赖版本 + 预期输出/验证法）  
- 失败分类或调试顺序  
- 面试追问 3–5 题（含参考答纲）  
- 本章自检（可勾选）  
- 关联 `skills[]` 与相关 Lab  

篇幅参考：主线核心章中文正文目标约 **2000–4000 字**（不含大段代码），避免再出现「500 字导览」。

### 5.2 Lab

每个必做 Lab：

- `examples/` 下独立可安装模块（或 monorepo 子包）  
- README：目标、能力 ID、本地真跑步骤、CI mock 说明  
- 验收清单 ≥5 条  
- 单测覆盖确定性逻辑；不在 CI 调付费模型  

**推荐 Lab 套件（主线）**

| Lab | 对应能力 | 主题 |
|---|---|---|
| L01 | S1 | 结构化输出与 Prompt 版本回归 |
| L02 | S2 | 订单只读 Tool 契约 + 非法参数 |
| L03 | S3/S4 | ReAct Agent + 流式事件 UI 契约 |
| L04 | S5 | RAG 入库/检索评估（Recall@K） |
| L05 | S6/S7 | Checkpoint + HITL 邮件确认 |
| L06 | S8 | Supervisor 双 Worker 最小权限 |
| L07 | S9 | 只读 MCP Server + 白名单 Agent |
| L08 | S10/S11 | 轨迹评估 + 输出护栏 + 基础 API 门禁 |

现有 `examples/knowledge-agent` 升级为作品集主载体（见第 6 节），不与 L04 简单重复：L04 偏检索评估，作品集偏端到端产品。

### 5.3 选修章

每选修章：问题背景、与主线差异、参考架构图、最小示例或阅读路径、面试追问。不强制与主线同等代码量，但必须有「能讲清楚」的验收。

## 6. 作品集封顶（最后做）

在能力 Lab 完成后升级 `examples/knowledge-agent`（或并列第二作品，默认先升级现有）：

- 私有知识问答 + 引用 + 拒答  
- 持久会话、HITL（至少一个副作用路径）  
- 评估集 ≥30、轨迹断言、注入样例  
- 部署/观测/成本说明  
- 演示脚本：成功 / 无证据 / 注入 / 跨 thread  

简历级交付物：架构图、评估报告、失败复盘、公开 README。

## 7. 站点与数据模型（实现默认，无需你拍板细节）

- 内容 `kind`: `lesson` | `lab` | `elective` | `capstone`  
- `skills: string[]` 挂能力地图  
- 侧栏按 kind 分组，组内可按 track 过滤  
- 能力地图页或大纲页：展示 S1–S11 / E1–E5 与完成进度（进度可先基于 localStorage）  
- 搜索覆盖课/Lab/选修正文与代码  
- 资源库保留，并与 skills 关联（次要）

## 8. 里程碑拆分（连续 PR，终局做满）

| 里程碑 | 交付结果（你可感知） | 审核 |
|---|---|---|
| M1 | 能力地图上线 + 内容模型/导航三层骨架 + 现有章迁移为 lesson | 独立 Claude 一次 |
| M2 | 加厚 S1–S4 课 + L01–L03 | 一次 |
| M3 | 加厚 S5–S8 课 + L04–L06 | 一次 |

流程：每里程碑 worktree → 功能分支 → 门禁 → PR → Claude 只审一次 → merge commit。  
Superpowers：每里程碑先有 plan，再实现。

## 9. 明确不做（防范围膨胀）

- 不做成视频平台或社区  
- 不在 CI 调用付费模型  
- 不镜像微信公众号未授权全文  
- 不把 Python 研究栈改为主线（选修可读，主线保持 TS/Node）  
- 不在单个 PR 塞完 M1–M5

## 10. 成功标准（上线后你怎么判断「够深」）

1. 任意主线能力 Si，能在 2 次点击内找到对应课 + Lab + 自检。  
2. 新人按 Lab README，在有 API Key 时 30–60 分钟跑通一个 Lab。  
3. CI 全绿且无付费 Provider 调用。  
4. 作品集 README 能单独作为面试作品说明。  
5. 抽查 3 章：均含反例、面试追问、可运行验证，不再是纯导览。

## 11. 风险与缓解

| 风险 | 缓解 |
|---|---|
| 「做满」周期长 | 严格按 M1–M5；每里程碑可独立上线 |
| 内容变长难维护 | 章/Lab 分文件；能力地图单源 |
| Lab 过重 | 每 Lab 单一能力；复用共享测试工具 |
| 与旧 16 章 URL | 保留 slug，迁移 kind，避免断链 |
